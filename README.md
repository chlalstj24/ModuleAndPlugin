# ModuleAndPlugin 프로젝트

언리얼 엔진의 프로젝트 구조(`Config`, `Content`, `Source`)를 파악하고, 독자적인 모듈과 플러그인을 구현하여 기능을 확장하는 방법을 익혔습니다.

## 주요 구현 기능

### 1. Test 모듈 (필수 기능)
- **목적**: 프로젝트의 기본 기능 외 별도의 C++ 모듈을 생성하고 프로젝트에 통합.
- **주요 내용**: 
    - `Test` 폴더 구성 및 빌드 설정(`Test.Build.cs`).
    - `TestActor` 구현 및 주 모듈(`ModuleAndPlugin`)에서 `SpawnActor`를 통해 동적 생성.

### 2. Temporary 플러그인 (필수 기능)
- **목적**: 재사용 가능한 플러그인 구조 수동 생성 및 프로젝트 연결.
- **주요 내용**:
    - `.uplugin` 메타데이터 구성.
    - 플러그인 내 모듈 인터페이스(`StartupModule`) 구현.

### 3. 캐릭터 데이터 관리 (도전 기능)
- **목적**: `UObject` 기반 클래스를 활용한 데이터 저장 및 클래스 간 참조 학습.
- **주요 내용**:
    - `UCharacterData` 클래스 구현(`UObject` 상속).
    - 캐릭터 블루프린트(`BP_MyCharacter`)에서 해당 데이터를 생성하고 초기값을 할당하여 화면에 출력.

## 학습 내용 및 문제 해결
- **모듈/플러그인 구조**: 언리얼의 계층 구조(솔루션-프로젝트-플러그인-모듈-소스코드)를 체계적으로 이해함.
- **에디터 연동 이슈**: 
    - C++ 클래스를 에디터에서 배치하거나 블루프린트로 생성하기 위해 `UCLASS(Blueprintable, BlueprintType)` 매크로와 `API` 매크로(e.g., `TEMPORARY_API`) 설정이 필수적임을 학습함.
    - 플러그인 내부 클래스가 블루프린트 생성 메뉴에 나타나지 않을 경우, `Intermediate/Binaries` 폴더를 삭제하고 `Rebuild`를 통해 UHT(Unreal Header Tool) 캐시를 초기화하는 과정을 통해 해결함.
